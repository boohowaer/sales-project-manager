# Claude 指令规范 (claude.md)

## 核心工作模式
- **语言偏好**：所有回复、注释、日志均使用 **中文**。
- **操作原则**：默认仅在**本地环境**进行代码修改、调试和文件操作。
- **环境隔离**：
  - 禁止擅自执行 `git push`。
  - 禁止擅自操作 GitHub 相关的操作。
  - 禁止擅自执行任何 `vercel` 部署命令。

## 部署工作流
只有当我明确输入指令 **“帮我发布”** 时，才按顺序执行以下操作：
1. **Git 提交**：将当前所有本地更改提交并推送到远端 Git/Github 仓库。
2. **Vercel 发布**：将项目部署/发布到 Vercel 上的项目 `sales-to-do`。
3. **状态确认**：发布完成后，提供 Vercel 的部署链接。
