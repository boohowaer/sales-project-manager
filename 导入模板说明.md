## 客户信息导入模板

### 文件格式要求
- **文件格式**: CSV (.csv) 或 Excel (.xlsx, .xls)
- **编码格式**: UTF-8（包含中文字符时请使用 CSV UTF-8 格式）
- **第一行**: 必须是字段名称（表头）

### 字段说明

| 序号 | 字段名称 | 是否必填 | 数据类型 | 填写格式说明 | 示例 |
|:---:|:---------|:--------:|:---------|:------------|:-----|
| 1 | **客户名称** | 必填 | 文本 | 最多200个字符，同一用户下不可重复，填写联系人姓名 | 陈明 |
| 2 | **公司名称** | 可选 | 文本 | 最多200个字符，填写客户所属公司 | 某某科技有限公司 |
| 3 | **邮箱** | 可选 | 邮箱 | 有效的邮箱格式 | contact@example.com |
| 4 | **电话** | 可选 | 文本 | 支持任意格式 | 13800001111 |
| 5 | **备注** | 可选 | 文本 | 最多5000个字符 | 重点客户 |

### 客户导入模板示例 (CSV格式)

```csv
客户名称*,公司名称,邮箱,电话,备注
陈明,某某科技有限公司,chenming@example.com,13800001111,重点客户，有长期合作意向
李华,ABC贸易有限公司,lihua@abc.com,13900002222,
王芳,DEF工业集团,wangfang@def.com,021-12345678,集团大客户
```

### 字段验证规则

1. **客户名称** (必填)
   - 填写**联系人姓名**
   - 不能为空
   - 长度: 1-200个字符
   - 同一用户下不能有重复的客户名称

2. **公司名称** (可选)
   - 长度: 0-200个字符

3. **邮箱** (可选)
   - 必须是有效的邮箱格式
   - 示例: contact@example.com

4. **电话** (可选)
   - 长度: 0-50个字符
   - 支持任意格式，系统原样保存

5. **备注** (可选)
   - 长度: 0-5000个字符
   - 可以为空

- **第一行**: 必须是字段名称（表头）


### 字段验证规则

1. **项目名称** (必填)
   - 不能为空
   - 长度: 1-200个字符

2. **客户名称** (必填)
   - 必须是系统中已存在的客户名称（联系人姓名）
   - 如果客户不存在，该行导入会失败

3. **客户来源** (可选)
   - 填写系统中「数据字典-客户来源」里配置的标签名称（如"线上推广"、"客户转介绍"）
   - 也支持填写字典的 key 值
   - 如果填写的值不在字典中，该字段会被忽略

4. **项目描述** (可选)
   - 长度: 0-5000个字符

5. **项目状态** (可选)
   - 可选值: `active` (跟进中), `won` (已成交), `lost` (已丢失), `on_hold` (暂停), `archived` (已归档)
   - 不填默认为 `active`
   - 大小写不敏感

6. **项目金额** (可选)
   - 必须是正数
   - 不含货币符号和千分位分隔符
   - 示例: `50000` (正确) vs `¥50,000` (错误)

7. **成功概率** (可选)
   - 0-100之间的整数
   - 不填默认为50

8. **开始日期** (可选)
   - 格式: YYYY-MM-DD 或 YYYY/M/D
   - 示例: 2026-01-15 或 2026/1/15

9. **预期成交日期** (可选)
   - 格式: YYYY-MM-DD 或 YYYY/M/D

10. **有开工函** (可选)
    - 可选值: `true`, `false`, `1`, `0`
    - 不填默认为 `false`

11. **已签署合同** (可选)
    - 可选值: `true`, `false`, `1`, `0`
    - 不填默认为 `false`

12. **归属年份** (可选)
    - 2000-2100之间的整数年份

13. **成交日期** (可选)
    - 格式: YYYY-MM-DD 或 YYYY/M/D

## 导入注意事项

### 通用注意事项

1. **文件编码**
   - 包含中文字符时请确保文件保存为 **UTF-8** 编码
   - Excel 用户请选择 "CSV UTF-8 (逗号分隔)(*.csv)" 格式

2. **字段分隔符**
   - CSV 文件使用**逗号**作为字段分隔符
   - 字段内容包含逗号时，用双引号包裹: `"某某科技,有限公司"`

3. **空值处理**
   - 可选字段不填写时留空即可
   - 示例: `项目A,客户名称,,,,` (正确)

4. **日期格式**
   - 推荐统一使用: `YYYY-MM-DD` 格式
   - 也支持 `YYYY/M/D` 格式

5. **数字格式**
   - 不要使用货币符号 (¥, $)
   - 不要使用千分位分隔符 (,)
   - 示例: `50000` (正确) vs `¥50,000` (错误)

6. **布尔值格式**
   - 推荐使用: `true`, `false`
   - 也支持: `1`, `0`
   - 大小写不敏感

### 客户导入注意事项

1. **客户名称唯一性**
   - 同一用户下不能有重复的客户名称
   - 如果导入的客户名称已存在，该行会导入失败

2. **邮箱格式**
   - 必须是有效的邮箱格式
   - 示例: contact@example.com (正确) vs contact@example (错误)

### 项目导入注意事项

1. **客户关联**
   - 导入项目前，必须先导入对应的客户
   - **客户名称**必须与系统中已存在的客户名称（联系人姓名）完全一致

2. **客户来源**
   - 填写系统中「数据字典-客户来源」配置的标签名称
   - 导入时系统会自动匹配字典，将标签名转换为对应的 key 值存储
   - 不在字典中的值会被忽略

3. **状态字段**
   - 项目状态使用英文代码:
     - `active` → 跟进中
     - `won` → 已成交
     - `lost` → 已丢失
     - `on_hold` → 暂停
     - `archived` → 已归档

4. **审批流程**
   - 销售代表 (`sales_rep`) 通过批量导入的数据会进入审批流程
   - 经理/管理员导入的数据直接入库，无需审批

### 导入流程建议

1. **第一步: 导入客户** — 先导入所有客户数据，确保客户名称准确
2. **第二步: 检查客户** — 登录系统确认客户已成功导入
3. **第三步: 导入项目** — 导入项目数据，客户名称需与系统一致
4. **第四步: 验证数据** — 检查项目列表，确认金额、日期、状态等字段

### 常见错误及解决方案

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| "客户名称不能为空" | 项目记录中客户名称为空 | 填写客户名称 |
| "客户不存在" | 客户名称在系统中找不到 | 先导入客户，或检查名称是否一致 |
| "邮箱格式无效" | 邮箱格式不正确 | 使用正确格式，如 contact@example.com |
| "日期格式无效" | 日期格式不正确 | 使用 YYYY-MM-DD 格式 |
| "金额格式无效" | 金额包含货币符号或千分位 | 移除 ¥ 和 ,，只保留数字 |
| "项目状态无效" | 状态值不在允许范围内 | 使用 active/won/lost/on_hold/archived |
