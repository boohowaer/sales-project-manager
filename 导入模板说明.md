## 👥 客户信息导入模板

### 文件格式要求
- **文件格式**: CSV (.csv) 或 Excel (.xlsx, .xls)
- **编码格式**: UTF-8 (如果包含中文字符，请使用CSV UTF-8格式)
- **第一行**: 必须是字段名称（表头）

### 字段说明

| 序号 | 字段名称 | 是否必填 | 数据类型 | 填写格式说明 | 示例 |
|:---:|:---------|:--------:|:---------|:------------|:-----|
| 1 | **客户名称** | ✅ 必填 | 文本 | 最多200个字符 | 张三 |
| 2 | **公司名称** | ⭕ 可选 | 文本 | 最多200个字符 | 某某科技有限公司 |
| 3 | **邮箱** | ⭕ 可选 | 邮箱 | 有效的邮箱格式 | customer@example.com |
| 4 | **电话** | ⭕ 可选 | 文本 | 支持任意格式的电话号码 | +86 138 0000 0000 或 138-0000-0000 |
| 5 | **备注** | ⭕ 可选 | 文本 | 最多5000个字符 | 客户的其他补充信息 |

### 📝 客户导入模板示例 (CSV格式)

```csv
客户名称,公司名称,邮箱,电话,备注
张三,某某科技,zhangsan@example.com,13800001111,重点客户
李四,ABC贸易公司,lisi@abc.com,13900002222,
王五,,wangwu@test.com,13600003333,个人客户，无公司
赵六,DEF工业,,,
```

### 字段验证规则

1. **客户名称** (必填)
   - 不能为空
   - 长度: 1-200个字符
   - 同一用户下不能有重复的客户名称

2. **公司名称** (可选)
   - 长度: 0-200个字符
   - 可以为空

3. **邮箱** (可选)
   - 必须是有效的邮箱格式
   - 长度: 0-200个字符
   - 示例: user@example.com

4. **电话** (可选)
   - 长度: 0-50个字符
   - 支持任意格式，系统会原样保存
   - 示例: +86 138 0000 0000, 138-0000-0000, (021) 1234-5678

5. **备注** (可选)
   - 长度: 0-5000个字符
   - 支持多行文本
   - 可以为空

- **第一行**: 必须是字段名称（表头）
- **重要**: 导入项目前，必须先确保对应的客户已经存在于系统中

### 字段说明

| 序号 | 字段名称 | 是否必填 | 数据类型 | 填写格式说明 | 示例 |

| 6 | **成功概率** | ⭕ 可选 | 整数 | 0-100之间的整数 (不填默认50) | 75 |

### 📝 项目导入模板示例 (CSV格式)

```csv
项目名称,客户名称,项目描述,项目状态,项目金额,成功概率,开始日期,预期关闭日期,有开工函,已签署合同,结算段数,归属年份
```

### 字段验证规则

1. **项目名称** (必填)
   - 不能为空
   - 长度: 1-200个字符
   - 同一客户下不建议有重复的项目名称

2. **客户名称** (必填)
   - 不能为空
   - 必须是系统中已存在的客户名称
   - 如果客户不存在，导入会失败

3. **项目描述** (可选)
   - 长度: 0-5000个字符
   - 支持多行文本
   - 可以为空

4. **项目状态** (可选)
   - 可选值: `active` (进行中), `won` (已成交), `lost` (已丢失), `on_hold` (暂停)
   - 不填默认为 `active`
   - 大小写不敏感

5. **项目金额** (可选)
   - 必须是正数
   - 保留两位小数
   - 范围: 0.00 - 99999999.99
   - 不含货币符号和千分位分隔符
   - 示例: `50000.00` 或 `50000` (正确) vs `¥50,000` (错误)

6. **成功概率** (可选)
   - 必须是0-100之间的整数
   - 不填默认为50
   - 示例: 0, 50, 75, 100

7. **开始日期** (可选)
   - 支持格式: YYYY-MM-DD 或 YYYY/M/D
   - 必须是有效的日期
   - 系统会自动转换为标准格式
   - 示例: 2024-01-15 或 2024/1/15

8. **预期关闭日期** (可选)
   - 支持格式: YYYY-MM-DD 或 YYYY/M/D
   - 必须是有效的日期
   - 建议晚于开始日期
   - 系统会自动转换为标准格式
   - 示例: 2024-06-30 或 2024/6/30

9. **有开工函** (可选)
   - 可选值: `true`, `false`, `1`, `0`
   - 不填默认为 `false`
   - 大小写不敏感

10. **已签署合同** (可选)
    - 可选值: `true`, `false`, `1`, `0`
    - 不填默认为 `false`
    - 大小写不敏感

11. **结算段数** (可选)
    - 必须是1-10之间的整数
    - 不填默认为1
    - 示例: 1, 2, 3, 4, 5

12. **归属年份** (可选)
    - 必须是2000-2100之间的整数年份
    - 用于统计和报表
    - 示例: 2024, 2025, 2026

## ⚠️ 导入注意事项

### 通用注意事项

1. **文件编码**
   - 如果数据包含中文字符，请确保文件保存为 **UTF-8** 编码
   - Excel用户请选择 "CSV UTF-8 (逗号分隔)(*.csv)" 格式

2. **字段分隔符**
   - CSV文件使用**逗号**作为字段分隔符
   - 如果字段内容中包含逗号，请用双引号包裹该字段
   - 示例: `张三,"某某科技,有限公司",zhangsan@example.com`

3. **空值处理**
   - 可选字段如果不填写，请留空或填写 `null`
   - 不要使用 `""` 或连续的逗号
   - 示例: `项目A,客户名称,,,,` (正确)

4. **日期格式**
   - 统一使用: `YYYY-MM-DD` 格式
   - 年份4位，月份2位，日期2位
   - 示例: 2024-01-15 (正确) vs 2024/1/15 (错误) vs 1/15/2024 (错误)

5. **数字格式**
   - 不要使用货币符号 (¥, $, €)
   - 不要使用千分位分隔符 (,)
   - 小数点使用点号 (.) 而非逗号 (,)
   - 示例: 50000.00 (正确) vs ¥50,000.00 (错误)

6. **布尔值格式**
   - 推荐使用: `true`, `false`
   - 也支持: `1`, `0`
   - 大小写不敏感: TRUE, True, true 都可以

### 客户导入注意事项

1. **客户名称唯一性**
   - 同一用户下不能有重复的客户名称
   - 如果导入的客户名称已存在，导入会失败

2. **邮箱格式**
   - 必须是有效的邮箱格式
   - 示例: user@example.com (正确) vs user@example (错误)

### 项目导入注意事项

1. **客户关联**
   - 导入项目前，必须先导入对应的客户
   - **客户名称**必须在系统中已存在
   - 如果客户不存在，该项目会导入失败

2. **状态字段**
   - 项目状态使用英文代码，不支持中文
   - 对应关系:
     - `active` → 进行中
     - `won` → 已成交
     - `lost` → 已丢失
     - `on_hold` → 暂停

3. **日期逻辑**
   - 预期关闭日期建议晚于开始日期
   - 系统不会强制验证，但建议保持逻辑一致性

4. **项目金额**
   - 单位: 元 (CNY)
   - 不含税
   - 精确到分 (两位小数)

### 导入流程建议

1. **第一步: 导入客户**
   - 先导入所有客户数据
   - 确保客户名称准确无误

2. **第二步: 检查客户**
   - 登录系统查看客户列表
   - 确认所有客户都已成功导入

3. **第三步: 导入项目**
   - 导入项目数据
   - 确保项目中的客户名称与系统中的客户名称完全一致

4. **第四步: 验证数据**
   - 检查项目列表
   - 验证金额、日期、状态等关键字段

### 常见错误及解决方案

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| "客户名称不能为空" | 项目记录中客户名称字段为空 | 检查CSV文件，填写客户名称 |
| "客户不存在" | 项目关联的客户名称在系统中找不到 | 先导入客户，或检查客户名称拼写是否一致 |
| "邮箱格式无效" | 邮箱字段格式不正确 | 使用正确的邮箱格式，如 user@example.com |
| "日期格式无效" | 日期格式不是 YYYY-MM-DD | 修改为正确的日期格式，如 2024-01-15 |
| "金额格式无效" | 金额包含货币符号或千分位分隔符 | 移除 ¥ 和 ,，只保留数字和小数点 |
| "项目状态无效" | 状态字段值不在允许的范围内 | 使用 active/won/lost/on_hold 之一 |
